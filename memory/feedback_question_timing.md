name: Question timing — give content before asking
description: User wants to read the full analysis/content before being asked to make a decision
type: feedback

Don't fire AskUserQuestion immediately after delivering content. Present the full analysis, options, or draft first — then ask the question only after the user has had time to read and digest.

**Why:** User said questions were coming too fast, before they had time to read the preceding content.

**How to apply:** After presenting a concept, analysis, or draft, end with a soft prompt like "Take your time — let me know when you're ready to decide direction" rather than an immediate AskUserQuestion. Only use AskUserQuestion when the user has indicated they're ready, or when the decision is truly blocking and minimal content preceded it.
